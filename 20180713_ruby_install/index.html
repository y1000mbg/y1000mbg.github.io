    ruby,Rails環境のセットアップ

    このページで分かること

    • UbuntuでRuby環境のセットアップ方法

    最近Progateを使ってRubyとRuby on Railsの勉強をしています。Progateは複雑な環境設定をしなくてもWeb上でコードやコマンドの動きを確認できるのが魅力ではありますが実際に自分で動かす環境構築は後々必須になってくるのでまとめました。

    環境

    • ホスト: Windows 10 Pro v1803
    • ゲスト: Ubuntu Ubuntu 16.04.4 LTS

    ※今回はvagrantのboxでUbuntuを用意してます。

    概要・主な流れ

    今回は下記を導入しrailsのwelcomeページが表示されるところまで環境を構築します。

    導入するもの 役割
    rbenv 複数のRubyを切り替えて使えるツール
    ruby-build rbenvを使うのに必要
    gem Railsの依存関係を解決してくれるツール。Rails自体のインストールやRailsで使うパッケージのインストールに使う
    Rails Webアプリケーションを作成できるフレームワーク

    Rubyの環境構築

    1. vagrantでUbuntuを用意
    2. rbenv ruby-build のインストール
    3. .bashrcに設定を書き込む
    4. rbenvでRubyをインストール

    Rubyの環境構築

    1. vagrantでUbuntuを用意

    Vagrantについては別記で構築方法を記載します。 コマンドプロンプトなどで任意のディレクトリを作成し下記コマンドを実行してくだささい。

    ※詳細はVagrantを使ってLinux開発環境を用意しようを御覧ください。

    1
2
    $c vagrant init ubuntu/xenial64
$c vagrant ssh

    2. rbenv ruby-build のインストール

    1
2
3
4
5
6
7
8
    ##c ~/.rbenvにインストールします。
$c cd ~/
$c git clone https://github.com/sstephenson/rbenv.git .rbenv
$c git clone https://github.com/sstephenson/ruby-build.git .rbenv/plugins/ruby-build

##c 必要パッケージのインストール
$c sudo apt update
$c sudo apt install -y ruby ruby-dev libsqlite3-dev nodejs autoconf bison build-essential gcc make libssl-dev libyaml-dev libreadline6-dev zlib1g-dev libncurses5-dev libffi-dev libgdbm3 libgdbm-dev

    ※1 libsqlite3-devが入ってないとrailsのプロジェクト作成時にエラーが出ます。

    ※2 nodejsが入ってないとrails server時にエラーが出て起動しません。

    3. .bashrcに設定を書き込む

    1
2
3
4
    $c echo "export RBENV_HOME=${HOME}/.rbenv" >> ~/.bashrc
$c echo -e "export PATH=\x22${HOME}/.rbenv/bin:\$PATH\x22" >> ~/.bashrc
$c echo 'eval "$(rbenv init -)"' >> ~/.bashrc
$c source ~/.bashrc
